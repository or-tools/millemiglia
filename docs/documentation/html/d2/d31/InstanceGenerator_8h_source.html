139 /*
140 * \brief Adds random shipments to a time-expanded transportation network.
141
142 * The network may be created using `make_random_network`, whose outputs can be
143 * passed as `state, network, distances`. This function adds shipments to the
144 * network by first sampling a shipment weight (from a truncated Pareto
145 * distribution [1]), a start node from the time expanded network, and then
146 * sampling trucks to make a shipment route. Routes end probabilistically after
147 * each truck to give an average route length of `mean_route_length`. The
148 * time-expanded network (`state`) should normally be pruned (using
149 * `prune_network` with `prune_shipments=False`) before adding shipments with this
150 * function. This method ensures that the network is completely solvable.
151 *
152 * \param rng: Random number generator.
153 * \param network: the logistic network where to add the shipments
154 * \param distances: Resistance distance matrix of `network`, as returned by
155 * `make_random_network`. This is used for sampling trucks in such a way that
156 * they are more likely to bring the shipment further from their start hub. How
157 * much this is taken into account is conrolled by the `dist_inv_temp` parameter.
158 * \param num_shipments: Number of shipments to be included in the network.
159 * \param mean_route_length: Average length of a shipment route (in time steps). After
160 * each time step of a shipment route, it is randomly decided whether to
161 * terminate the route, with probability 1 / `mean_route_length`. This way, a
162 * shipment route lasts `sum((1 - 1 / mean_route_length)**k)` time steps in
163 * expectation. This geometric sum approaches `mean_route_length` as the
164 * number of time steps in `state` becomes larger.
165 * \param min_shipment_weight: The scale (`m`) parameter of the Pareto distribution [1]
166 * from which shipment weights are sampled.
167 * \param max_shipment_weight: The maximum weight of shipments. The shipment weights are
168 * sampled from a truncated Pareto distribution using rejection sampling to
169 * keep weights below `max_shipment_weight`.
170 * \param shipment_weight_shape: The shape (`a`) parameter of the Pareto distribution
171 * [1] from which shipment weights are sampled.
172 * \param unit_weights: If `True`, all shipment weights are set to 1 instead of being
173 * sampled from a Pareto distribution.
174 * \param start_inv_temp: The start node hub is sampled from a Boltzmann distribution
175 * with exponent `-start_inv_temp * degree(hub)` to preferably start at hubs
176 * with lower degree.
177 * \param dist_inv_temp: When sampling shipment routes it is often posssible to choose
178 * between multiple trucks. In these cases, the truck is chosen among the
179 * available trucks by sampling from a Boltzmann distribution with exponent
180 * `dist_inv_temp * dist_from_start` (where the distance to the start node is
181 * measured in terms of resistance distance, i.e. `distances`). This
182 * encourages shipments to go further from their start nodes.
183 * \param max_tries: If, after a shipment route has been sampled, this shipment route
184 * doesn't actually use any real trucks (instead, the shipment just stays at
185 * its start hub), we try again. This is done for a maximum of `max_tries`
186 * times (after which a 'staying' shipment route will simply be accepted). The
187 * shipment weight is reduced by 10% each try to make it easier to place the
188 * shipment.
189 * \param cut_capacities: After all shipments have been placed in the network, there is
190 * normally still capacity left in some trucks. `cut_capacities` is a number
191 * between 0 and 1 that specifies how much of this excess capacity should be
192 * removed (by reducing the trucks' capacities). Thus, 1 means all excess
193 * capacity is cut and 0 means no capacity is cut.
194 *
195 * The new state (time-expanded network), now with `num_shipments` shipments added,
196 * as well a dictionary mapping each shipment id to a list of nodes (in
197 * (location, time) format) representing a solution route for that shipment.
198 */
199 void generate_shipments(const operations_research::lattle::LogisticsNetwork& network, const SpaceTimeNetwork& st_network, const int& shipment_number,
200 const int& timesteps, const int& mean_path_length, const int& min_shipment_weight = 1, const int& max_shipment_weight = 100,
201 const double& shipment_weight_shape = 0.1, const bool& unit_weights = false, const double& start_inv_temp = 0.1, const double& dist_inv_temp = 0.1,
202 const int& max_tries = 50, const double& cut_capacities = 0.0) const;
